# 投资晨报网页部署说明

## 架构

- Vercel：托管单页网页并每天触发生成任务；
- 公开RSS：提供新闻候选；
- GitHub Models：提供限额免费的AI分析；
- GitHub：保存当前晨报和所有历史晨报。

## 每日自动流程

北京时间目标08:00，Vercel调用 `/api/cron`。程序并行抓取RSS，让AI按既定结构生成晨报，校验数量与来源后依次写入：

1. `data/history/YYYY-MM-DD.json`：完整归档；
2. `data/history.json`：历史目录；
3. `data/today.json`：最新一期。

网页直接读取GitHub中的最新数据，不需要每天重新构建。任一步失败时，上一期仍然可读。

## 必需环境变量

- `CRON_SECRET`：保护定时接口的随机长字符串；
- `GH_MODELS_TOKEN`：只授予GitHub Models读取权限；
- `GH_CONTENT_TOKEN`：只授予当前仓库Contents读写权限；
- `GH_CONTENT_REPOSITORY`：`2538449420-cloud/investment-morning-report`；
- `AI_PROVIDER`：`github_models`；
- `AI_MODEL`：账号可用的免费模型名称。

所有变量只配置在Vercel Production环境，不能写入代码或聊天。

## 上线步骤

1. 使用GitHub登录Vercel；
2. 导入 `investment-morning-report`；
3. Framework选择Other，默认设置完成首次部署；
4. 添加上述环境变量；
5. 重新部署；
6. 查看Cron日志并进行首次测试；
7. 打开网页的“往期晨报”，确认归档可读。

## 边界

免费模型具有额度与频率限制，不能保证永久免费；Vercel免费定时任务也可能延迟几分钟。系统不会在失败时编造内容或删除上一期。以后可仅通过环境变量切换到其他兼容AI接口。
