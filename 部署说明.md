# 投资晨报个人版部署说明

## 推荐方案

使用 GitHub Pages 负责在线阅读，使用 GitHub Actions 每天自动生成和发布晨报。

优点：

- 电脑关机后也能运行；
- 不需要长期维护一台服务器；
- API密钥保存在GitHub加密密钥中，不进入网页代码；
- 每日晨报和知识树可以保留历史版本；
- 可以先作为个人网页使用，以后再接微信小程序。

## 自动时间

工作流在北京时间07:30开始搜索、生成和检查，目标是在08:00前完成页面更新。云端定时任务偶尔可能排队，因此不能把发布时间保证到秒。

## 上线需要的两个账号

1. GitHub账号：保存项目并发布网页；
2. OpenAI API账号：提供联网搜索和晨报生成能力。

## 安全规则

- 不要把API密钥写入代码、网页或聊天记录；
- 只在GitHub仓库的 `Settings → Secrets and variables → Actions` 中添加；
- 密钥名称固定为 `OPENAI_API_KEY`；
- 仓库中只会出现 `${{ secrets.OPENAI_API_KEY }}`，不会出现真实密钥。

## 已准备好的部署文件

- `.github/workflows/daily-report.yml`：每日生成与发布；
- `.github/workflows/deploy-preview.yml`：首次手动发布；
- `launcher/generate_once.py`：单次晨报生成器；
- `data/latest.json`：在线页面读取的最新晨报；
- `data/history/`：后续保存历史晨报；
- `data/knowledge.json`：轻量知识树记忆。

## 下一步

登录GitHub，创建一个仓库，将本文件夹上传，然后在仓库设置中启用GitHub Pages并配置API密钥。首次手动发布成功后，每日工作流会自动接管。
